Uploading Data to LabCAS

This document provides instructions on how to upload data into LabCAS with information on:

  • Generating your public and private keys
  • Getting your SFTP username
  • Installing and running the SFTP software

Before uploading data, please make sure to organize your files in a directory structure corresponding to a collection consisting of one or more datasets, as previously determined with the JPL Informatics Center.

SFTP: The SSH File Transfer Protocol (one-time setup)

To upload data into LabCAS, you‚Äôll use the SSH (Secure Shell) File Transfer Protocol (SFTP), which requires:

  1. A pair of key files:
    • A private key file known only to you.
    • A public key you send to the JPL Informatics Center via email.
  2. An SFTP username. The JPL Informatics Center will assign this to you when we get your public key file.

üö® You should keep your private key file safe! It‚Äôs essentially a password, and disclosing it could put your data at risk.

To make your key files:

  1. Visit the key generator website (link opens in a new tab).
  2. Leave the ‚Äúpassword‚Äù field blank.
  3. Set the ‚Äútype‚Äù to rsa (the default).
  4. Set the ‚Äúlength‚Äù to 4096.
  5. Press Create Key. You now have a new pair of keys.
  6. Under the box labeled ‚ÄúPrivate Key‚Äù, press Download and save this file. This is your private key which you must keep safe! The file will be named file.key but feel free to rename it something more useful like labcas-private-key.
  7. Under the box labeled ‚ÄúPublic Key‚Äù, press ‚ÄúCopy to Clipboard‚Äù and paste this into an email to the JPL Informatics Center. Your email must include:
    • Your newly generated public key (pasted in)
    • Your name
    • Your institution‚Äôs name
    • The name of the project for which you‚Äôll be uploading data, such as ‚ÄúEDRN Prostate MRI‚Äù or ‚ÄúMCL Pre-Cancerous Lesions‚Äù

üëâ Note: The private key file needs to be on the machine which has the data you want to upload. If you generated the keys on your laptop computer, for example, but the science data is on another computer, copy the private key from the laptop to the other system. To be extra safe, delete the private key from the original system!

Once the JPL team processes your email, you‚Äôll be sent your SFTP username and you‚Äôll be ready to upload data.

üëâ Note: Your SFTP username may not be the same as your EDRN, MCL, NCI, or institutional username‚Äîor even the username you use to log into your computer.

Unix Users (including Apple macOS)

If you use Apple macOS, some kind of Linux, or another flavor of Unix, then follow the instructions below. Windows users, please see the next section.

Installing your Software and Private Key (one-time setup‚Äîon the machine you will be using to upload files)

Most modern Unix systems already include the sftp program. To see if it‚Äôs on yours, open a terminal window and type sftp and press Return or Enter. If you get a ‚Äúusage‚Äù message, you‚Äôve got sftp. If not, ask your system administrator to have sftp installed.

Next, locate the private key file you downloaded from the steps above. It could be in your Downloads folder, your Documents folder, or in another location. For example, if you‚Äôre on macOS and your computer‚Äôs login name is lisa and your private key file is file.key and you save the file in your Downloads folder, then the full path to your private key file is:

/Users/lisa/Downloads/file.key

You‚Äôll need the full path to your private key file (as well as your SFTP username) for the next section.

üö® Important! The private key file must be readable only by you. Type the following command in a terminal window to make sure this is the case: chmod 600 PRIVATEKEYFILE.key. For example, if your private key file is in /home/tgreene/Desktop/cancer-institute.key, type:

chmod 600 /home/tgreene/Desktop/cancer-institute.key

Running SFTP (Uploading your data)

Connect with sftp. Open a terminal and change the current working directory to where you have your collection of data to upload. Then enter the following command:

sftp -C -i PRIVATEKEYFILE.key USERNAME@labcas.ftpaccess.cc

Replace PRIVATEKEYFILE.key with the full path to your private key file and replace USERNAME with your assigned SFTP username. For example, if your SFTP username was assigned by the JPL Informatics Center as lisa-holmes and your private key file is in /Users/lisa/Documents/secret.key, the command you would enter is:

sftp -C -i /Users/lisa/Documents/secret.key lisa-holmes@labcas.ftpaccess.cc

üëâ Note: The first time you run this command, you may see this message‚Äîor something similar to it:

The authenticity of host 'labcas.ftpaccess.cc (44.236.254.100)' can't be established.
RSA key fingerprint is SHA256:6xtOnepeFGgKqrN3gR7s9hAdQo98LUEKKtzjg1B1dYo.
Are you sure you want to continue connecting (yes/no)?

Please answer yes to this.

Running this command will put you at the sftp> prompt. From this prompt enter the following command:

put *

This will place all local files in the current local directory into the current directory on the server.

Other useful commands (if needed):

  • Create a sub-directory on the server: mkdir SUBDIR
  • Change into that sub-directory on the server: cd SUBDIR
  • Put all local files recursively (in sub-directories) in the current local directory into the current directory on the server: put -r * . (note the final dot or period at the end of this command).

Once you‚Äôve put all your files onto the LabCAS server, email the Informatics Center to let them know the new data is available and to coordinate the publishing phase.

Windows Users

If you use the Windows operating system, follow these instructions. Unix (including Apple macOS) users, see the preceding section above.

Installing your Software and Private Key (one-time setup)

The Windows operating system doesn‚Äôt come with an SFTP program. The program we recommend is WinSCP. As of this writing, the latest version of WinSCP is 5.19.5.

To get WinSCP:

  1. Open the Microsoft Store on your computer.
  2. Type ‚ÄúWinSCP‚Äù in the search box.
  3. Click ‚ÄúBuy‚Äù.

You can also get WinSCP for free by visiting the WinSCP download page, scrolling past the advertisements, and clicking the ‚ÄúDOWNLOAD WINSCP‚Äù (with version number) button at the bottom.

In both cases, you may need administrator privileges to complete the installation. The default choices for various installation options are fine in all cases.

Next, locate the private key you generated using the steps above on the machine you will be using to upload files,such as your ‚ÄúDocuments‚Äù folder, or on your Desktop. You‚Äôll need that below.

Running WinSCP

With your SFTP username, private key file, and WinSCP, you‚Äôre ready to upload data. Start by launching WinSCP from the Windows menu or from its application icon. On the first launch, a window will appear where you can enter new login information. Enter the information as follows:

  • Host name: labcas.ftpaccess.cc
  • Port number: 22
  • User name: Enter your SFTP username assigned by the JPL Informatics Center
  • Password: leave this blank

Next, click the ‚ÄúAdvanced‚Äù button so you can tell WinSCP where your private key file is:

  1. In the left panel, click ‚ÄúEnvironment‚Äù and make sure ‚ÄúPreserve remote timestamp‚Äù is not selected; select any of the other two options if it is.
  2. In the left panel, click ‚ÄúAuthentication‚Äù under ‚ÄúSSH‚Äù.
  3. Click the ‚Ä¶ (ellipsis) button.
  4. In the lower-right, click the file filter drop-down and select ‚ÄúAll Files (*.*)‚Äù.
  5. Navigate to your saved private key file and select it.
  6. Click ‚ÄúOpen‚Äù.
  7. When prompted ‚ÄúDo you want to convert this OpenSSH private key to PuTTY format?‚Äù click ‚ÄúOK‚Äù.
  8. When prompted to ‚ÄúSave converted private key‚Äù, click ‚ÄúSave‚Äù. This saves the private key in a format compatible with WinSCP in the same folder as the private key you downloaded from the key generator website, but with a new extension of .ppk instead of .key.
  9. Click ‚ÄúOK‚Äù on the confirmation dialog indicating the key was converted and saved.
  10. Click ‚ÄúOK‚Äù to close the ‚ÄúAdvanced Site Settings‚Äù.
  11. Click ‚ÄúSave‚Äù to save all these settings for future use; click ‚ÄúOK‚Äù to use the default save settings.
  12. Finally, click ‚ÄúLogin‚Äù to connect to the LabCAS SFTP server.

The first time you connect, you may be asked ‚ÄúContinue connecting to an unknown server and add its host key to a cache?‚Äù; please click Yes to this.

At this point, your WinSCP window displays folders on the LabCAS server. You can then simply upload your files to the LabCAS server by dragging and dropping them to the appropriate sub-directory.

üëâ Note: You may see an error message that mentions something like the following:

Upload of file‚Ä¶was successful but error occurred while setting the permissions and/or timestamp.

If this appears, please just click the Skip all button and allow WinSCP to continue normally.

Once you‚Äôve put all your files onto the LabCAS server, email the Informatics Center to let them know the new data is available and to coordinate the publishing phase.

Metadata and extensions

Adding metadata to LABCAS

For a single file and/or directory:

Cfg format:

  • Create a file with .cfg extension. The file should follow the following file name pattern: <file/directory name>.cfg
  • If adding metadata for a file named "C001.dcm", the metadata file will be "C001.dcm.cfg" and in the same directory as the file.
  • If adding a metadata for a directory named "Images", the metadata file will be "Images.cfg" and located directly under the "Images" directory.
  • Each line should contain one key value pair separated by an "=".
  • There should not be any duplicates for the keys.
  • Cfg file format does not allow multi-line values.

   

XmlMet format:

  • Create a file with .xmlmet extension. The file should follow the following file name pattern: <file/directory name>.xmlmet
  • If adding metadata for a file named "C001.dcm", the metadata file will be "C001.dcm.xmlmet" and in the same directory as the file.
  • If adding a metadata for a directory named "Images", the metadata file will be "Images.xmlmet" and located directly under the "Images" directory.
  • The xml file should have the following format:

                            <labcas:metadata>

                            <keyval>

                    <key>Field_1</key>

                    <val>Value_1</val>

                            </keyval>

                            <keyval>

                    <key>Field_2</key>

                    <val>Value_2</val>

                            </keyval>

                            </labcas:metadata>

Excel, Csv and Txt format:

  • Create a file with .excel, .csv or .txt (tab separated file) extension. The file should follow the following file name pattern: <file/directory name>_META.excel/csv/txt
  • If adding metadata for a file named "C001.dcm", the metadata file will be "C001.dcm_META.excel/csv/txt" and in the same directory as the file.
  • If adding a metadata for a directory named "Images", the metadata file will be "Images_META.excel/csv/txt" and located directly under the "Images" directory.
  • Make sure there is one row with all the metadata and a column, either called FileName with the exact name of that file or directory, or called LABCAS_ID with the LabCAS ID of that file or directory.

For multiple files and/or directories:

Excel, Csv and Txt format:

  • Create a file with .excel, .csv or .txt (tab separated file) extension. The file should follow the following file name pattern: *_META.excel/csv/txt, or let the LABCAS managers know the exact name of the file beforehand.
  • Make sure it is either in the same location as the files or directories or in a parent directory.
  • Make sure there is a row per file and/or directory with all the metadata and a column, either called FileName with the exact name of the file or directory, or called LABCAS_ID with the LabCAS ID of the files or directories.

Metadata Inheritance:

  • LABCAS allows the "flow" of metadata up and down the directory structure or hierarchy. So if you want to apply some metadata to all the files of a dataset or collection, you just need to add that metadata to that dataset or the collection.
  • On the flip side, if we have say, a "FileType" metadata at the file level, and we have multiple types of files. This metadata will also show up at the Collection and Dataset levels with the value as an aggregated, deduplicated list of all file types available under that Dataset and/or Collection.
  • The upward and downward flow of metadata can be fine tuned by limiting the flow of certain metadata fields by adding them to a LABCAS config file, which can only be done by a LABCAS manager.
